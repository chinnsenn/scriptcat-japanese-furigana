# Japanese Furigana for Web Pages

## 安装

1. 打开 [Greasy Fork 数字链接](https://greasyfork.org/scripts/587522) 安装脚本。
2. 通过脚本菜单中的“设置 Yahoo Client ID”输入自己的 Yahoo! JAPAN Developer Network Client ID。
3. 打开任意日语页面，页面边缘会出现圆形抽象注音图标。
4. 首次远程分析时核对站点、范围、字符数和摘要，再确认当前站点许可。

也可以通过 ScriptCat 脚本菜单中的“设置 Yahoo Client ID”提前配置。

## 工作方式

- 脚本通过页面的 `lang` 属性和可见假名比例识别日语页面。
- 点击按钮后，脚本按段落提取可见文本并发送到 Yahoo! JAPAN ルビ振り API。
- 可通过“设置默认标注范围”菜单选择选中文本、正文区域或整页。
- 正文发送许可按当前网站 origin 独立保存，可通过“撤销当前站点正文发送许可”菜单清除。
- 自动标注白名单按 origin 独立保存，可通过“切换当前站点自动标注”菜单管理。
- 首次许可会展示实际范围与首个未缓存片段摘要；“查看本次实际发送范围”只列出真正进入远程请求的文本摘要。
- 返回的读音通过标准 `<ruby><rt>` 元素写回页面；跨内联元素词语使用一个连续 ruby，并保留原元素身份与事件。
- 圆形按钮用米白、金色进度环和松绿色表达待标注、处理中和已完成；按钮内部无文字，动态 `aria-label` 提供完整状态。
- 每个文本块优先读取当前页面域名下的 `localStorage` 缓存，内容变化、缓存过期或缓存缺失时才调用 API。
- 缓存保留 30 天、最多 200 个文本块；内存中另保留最近使用的 300 个结果。
- 光标悬停按钮会显示进度、标注数、API 额度与调用、缓存命中、分析量、限流等待、重试、失败片段和最近耗时。
- 按钮尺寸为 `48 × 48px`，支持鼠标和触摸拖动；松手后吸附最近边缘，并跨页面保存边缘和相对位置。
- 开启期间会把 SPA 导航、无限滚动、节点增删、文本改写和局部重渲染加入脏区队列，分析中的新变化会在下一批继续处理。
- 远程调用达到每分钟 300 次窗口上限时会等待额度恢复；Yahoo 429 响应会按 `Retry-After` 自动重试一次。
- 网络、超时与 5xx 瞬时错误会指数退避重试两次；最终失败片段保留其余结果，并可通过“重试失败片段”菜单再次处理。

## 隐私

脚本会把所选范围中的合格可见正文分段发送到 `jlp.yahooapis.jp`。采集会跳过导航、页眉页脚、侧栏、表单、代码块、可编辑区域、隐藏区域和已有 ruby。请在公开文章等合适页面使用。

Client ID 保存在 ScriptCat 的脚本存储中。发布到脚本市场时，应让每位使用者配置自己的 Client ID。

“清理当前站点读音缓存”会同时删除当前页面引擎的内存条目和当前 origin 的 `localStorage` 条目；站点许可与 Client ID 保持独立。

脚本从 0.1.1 起使用 Yahoo 官方支持的 `appid` 参数传递 Client ID，适配浏览器扩展环境。
0.1.2 会在 Yahoo 返回 `Invalid params` 时自动缩小文本块重试；单个异常片段达到最小粒度后会被跳过，其余正文继续标注。
0.1.3 将 Yahoo 响应缓存限制为最近使用的 300 个文本块，长时间浏览和 SPA 增量标注时会自动淘汰旧结果。
0.1.4 在按钮中显示本页脚本最近 60 秒的估算剩余额度，例如 `标注读音 · 298/300`。Yahoo 的限制按 Client ID 汇总，其他标签页、设备和应用产生的请求不会出现在本页计数中。
0.2.0 将按钮收束为两个状态，把额度移入悬浮统计面板，并增加按文本内容校验的 `localStorage` 持久缓存。
0.2.1 缩小按钮并增加拖动、最近边缘吸附、位置持久化和窗口缩放自适应。
0.3.0 增加站点级正文发送许可、真实请求限流、429 退避、脏根增量处理与会话指标重置。
1.0 Sprint 01 增加三种采集范围、页面外壳排除与保留元素身份的跨节点连续 ruby。

## 已知边界

- API 单次请求上限为 4KB；脚本使用约 3KB 的保守分块。
- 跨节点词元需要在公共祖先下形成连续安全区间；混入被排除内容的异常区间会跳过。
- 人名、地名和作品自定义读音仍可能存在歧义。
- 页面框架重新渲染后，脚本会尝试增量恢复注音。
- `localStorage` 按网站域名隔离；禁用或容量不足时会自动退回内存缓存与 API。
