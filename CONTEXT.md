# 日语网页注音

本项目在网页正文与上下文相关读音之间建立可逆转换，同时把远程分析、页面变更和用户会话保持为独立概念。

## Language

**读音分析**:
把一组日语正文转换为带文本区间的上下文相关汉字读音。
_Avoid_: API 分析、Yahoo 处理、分词

**页面标注**:
把读音区间写成可完整撤销的网页 `ruby` 结构。
_Avoid_: DOM 替换、页面翻译

**注音会话**:
从用户开启标注到撤销标注期间的状态、统计和动态正文处理周期。
_Avoid_: 按钮状态、运行任务

**远程授权**:
允许指定站点 origin 的缓存缺失正文使用 Client ID 发送至读音提供方的用户许可。
_Avoid_: 登录、OAuth、API 开关

**失败片段**:
一次读音分析中经过重试仍无法完成的最小正文范围；它独立保留来源段落与偏移，供当前注音会话重试。
_Avoid_: 整页失败、异常文本

**发送审计**:
当前页面生命周期内真正进入远程请求的采集范围、次数、字符数与受限文本摘要。
_Avoid_: 待分析正文、缓存内容、历史记录

## Relationships

- 一次 **注音会话** 包含一次或多次 **读音分析**。
- 每次 **读音分析** 产生零个或多个读音区间，交给 **页面标注** 写入。
- 缓存未命中的 **读音分析** 必须获得 **远程授权**。
- **远程授权** 只覆盖用户确认的站点 origin，可独立撤销。
- **失败片段** 不阻断同批次其他读音区间，可在原页面标注位置独立重试。
- **发送审计** 只记录缓存未命中后发生的真实请求，页面关闭后自然消失。
- **页面标注** 的撤销会结束当前 **注音会话**。

## Example dialogue

> **Dev:** “动态加入的新段落需要重新开始注音会话吗？”
> **Domain expert:** “继续当前注音会话，只对新增正文执行读音分析，再增量页面标注。”

## Flagged ambiguities

- “标注”曾同时表示读音计算和 DOM 写入；现固定为 **读音分析** 与 **页面标注** 两个阶段。
